# 积分系统设计文档 v2.1

> 统一积分管理，支持完整追溯 | 更新日期: 2026-03-26

---

## 设计原则

### 1. 积分日志为权威数据源
- `point_logs` 表的 SUM(amount) 作为权威余额
- `users.stars` 和 `users.total_stars` 作为缓存，应与 point_logs 保持同步
- 读取余额时从 point_logs 实时计算，保证数据一致性

### 2. 完整追溯
- `point_logs` 表记录所有积分变动的完整流水
- 每条记录包含：用户、变动数量、变动后余额、类型、来源、描述、时间

### 3. 事务保证
- 所有积分变更都在数据库事务中完成
- 使用 `FOR UPDATE` 锁定用户记录防止并发问题

### 4. 一致性验证
- 提供每日自动核查脚本（凌晨4点执行）
- 发现不一致时自动修复
- 日志文件：`/opt/xuexi-app/logs/points_consistency.log`

## 数据库结构

### users 表（积分缓存）

| 字段 | 类型 | 说明 |
|------|------|------|
| `stars` | INTEGER | 当前可用余额（与point_logs同步） |
| `total_stars` | INTEGER | 历史累计获得（与point_logs同步） |
| `version` | INTEGER | 乐观锁版本号 |

### point_logs 表（积分流水 - 权威数据源）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键 |
| `user_id` | UUID | 用户ID |
| `amount` | INTEGER | 变动数量（正=获得，负=消耗） |
| `balance_after` | INTEGER | 变动后的余额 |
| `type` | VARCHAR(30) | 变动类型 |
| `source` | VARCHAR(50) | 来源模块 |
| `source_id` | UUID | 来源记录ID |
| `description` | VARCHAR(200) | 变动描述 |
| `created_at` | TIMESTAMP | 变动时间 |

### 积分类型说明

| Type | 说明 | 方向 |
|------|------|------|
| `signin` | 每日签到 | 正 |
| `task_approve` | 任务完成审批通过 | 正 |
| `achievement` | 成就奖励 | 正 |
| `wheel` | 转盘中奖 | 正 |
| `exchange` | 兑换奖励 | 负 |
| `lottery` | 贴纸抽奖 | 负 |
| `deduct` | 惩罚扣分 | 负 |
| `adjust` | 手动调整 | 正/负 |

## 积分获取与消耗

### 获得积分方式
1. 完成任务（家长审批后）+N 星星
2. 每日签到（连续天数递增奖励）
3. 解锁成就（一次性奖励）
4. 转盘中奖（随机1-5星）

### 消耗积分方式
1. 兑换奖励（需家长审批）
2. 贴纸抽奖（5星/次，20次保底）
3. 宠物照顾（消耗星星）
4. 惩罚扣分（家长手动）

### point_transaction_types 表（积分类型定义）

| type | name | direction | 说明 |
|------|------|-----------|------|
| signin | 每日签到 | earn | 每日签到获得的积分 |
| task_approve | 任务完成 | earn | 完成任务并通过审批获得的积分 |
| achievement | 成就奖励 | earn | 解锁成就获得的积分 |
| wheel | 转盘中奖 | earn | 转盘抽奖获得的积分 |
| lottery | 贴纸抽奖 | spend | 贴纸抽奖消耗的积分 |
| exchange | 兑换奖励 | spend | 兑换奖励消耗的积分 |
| pet_care | 宠物照顾 | spend | 照顾宠物消耗的积分 |
| deduct | 惩罚扣分 | spend | 家长扣分惩罚消耗的积分 |
| adjust | 手动调整 | neutral | 管理员手动调整 |
| reverse | 撤销返还 | earn | 撤销审批后返还的积分 |

## 统一积分服务 API

位置: `/backend/src/services/points.js`

### addPoints(userId, amount, pointType, options)

增加用户积分

```javascript
const result = await addPoints(userId, 10, PointType.SIGNIN, {
  sourceId: logId,
  description: '签到获得 10 星星'
})

// 返回
{
  success: true,
  balance: 15,    // 当前余额
  earned: 10       // 本次获得
}
```

### subtractPoints(userId, amount, pointType, options)

扣除用户积分

```javascript
const result = await subtractPoints(userId, 5, PointType.EXCHANGE, {
  sourceId: exchangeId,
  description: '兑换奖励消耗 5 星星'
})

// 返回
{
  success: true,
  balance: 10,    // 当前余额
  spent: 5         // 本次消耗
}

// 余额不足时
{
  success: false,
  error: '余额不足',
  required: 10,
  available: 5
}
```

### getPointsInfo(userId)

获取用户积分信息

```javascript
const info = await getPointsInfo(userId)

// 返回
{
  success: true,
  balance: 100,        // 当前余额
  totalEarned: 150,    // 累计获得
  totalSpent: 50,      // 累计消耗
  earnCount: 20,       // 获得次数
  spendCount: 10,       // 消耗次数
  isConsistent: true,  // 数据是否一致
  difference: 0         // 不一致的差值
}
```

### getPointsHistory(userId, options)

获取积分变动历史

```javascript
const history = await getPointsHistory(userId, {
  limit: 50,
  offset: 0,
  type: 'signin'  // 可选，筛选类型
})

// 返回
{
  success: true,
  items: [...],      // 变动记录列表
  total: 100,         // 总数
  hasMore: true      // 是否有更多
}
```

### fixPointsConsistency(userId)

修复单个用户的积分一致性

### fixAllPointsConsistency()

批量修复所有用户的积分一致性

## 使用示例

### 在签到模块中使用

```javascript
import { addPoints, PointType } from '../services/points.js'

// 签到逻辑
const bonus = calculateSigninBonus()
const result = await addPoints(userId, bonus, PointType.SIGNIN, {
  description: `签到获得 ${bonus} 星星（连续${streak}天）`
})
```

### 在审批任务模块中使用

```javascript
import { addPoints, PointType } from '../services/points.js'

// 审批通过后
const result = await addPoints(userId, starsEarned, PointType.TASK_APPROVE, {
  sourceId: taskLogId,
  description: `完成任务「${taskTitle}」获得 ${starsEarned} 星星`
})
```

### 在兑换模块中使用

```javascript
import { subtractPoints, PointType } from '../services/points.js'

// 兑换奖励
const result = await subtractPoints(userId, cost, PointType.EXCHANGE, {
  sourceId: exchangeId,
  description: `兑换「${rewardTitle}」消耗 ${cost} 星星`
})

if (!result.success) {
  // 余额不足处理
  ctx.body = error(3003, '余额不足')
  return
}
```

## 扩展指南

### 添加新的积分类型

1. 在 `point_transaction_types` 表中添加新类型
2. 在 `PointType` 枚举中添加常量
3. 在相关模块中调用 `addPoints` 或 `subtractPoints`

```javascript
// 示例：添加签到奖励类型
export const PointType = {
  // ... 其他类型
  SIGNIN_BONUS: 'signin_bonus'  // 连续签到奖励
}
```

### 查询积分统计

```javascript
// 按类型统计
const stats = await pool.query(`
  SELECT type, COUNT(*) as count, SUM(amount) as total
  FROM point_logs
  WHERE user_id = $1
  GROUP BY type
`, [userId])
```
